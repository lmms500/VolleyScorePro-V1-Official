# PRD: Redesign do MatchOverModal (Neo-Glass 3.0)

## Contexto
Redesign completo do componente `MatchOverModal` para elevar a interface ao padrão "Premium/AAA", alinhando-se com a referência visual fornecida (screenshot). O objetivo é melhorar a hierarquia visual, clareza das informações de rotação e consistência com o Design System Neo-Glass 3.0 do projeto.

## Arquivos Afetados
*   `src/components/modals/MatchOverModal.tsx`
*   `src/index.css` (para classes utilitárias de glassmorphism, se necessário)

## Plano de Ação

### 1. Estrutura Visual e Layout
O modal será dividido em 4 seções principais, utilizando Flexbox para alinhamento vertical:

1.  **Header (Vencedor & Placar)**
    *   Ícone de Troféu (`Trophy`) centralizado e com brilho (drop-shadow).
    *   Nome do Time Vencedor em destaque (Caixa alta, Weight: Black).
    *   Placar Final (Sets) com números grandes e separador sutil.

2.  **Seção "Próxima Rotação"**
    *   **Título:** "PRÓXIMA ROTAÇÃO" centralizado, flanqueado por linhas horizontais (`h-px bg-white/10`).
    *   **Cards de Troca (Container Horizontal):**
        *   **Card Saindo (Esquerda):** Fundo Glass escuro com tintura vermelha/roxa.
            *   Ícone: `LogOut` (Lucide) em badge pequeno.
            *   Texto: "SAINDO" + Nome do Time.
            *   Contador: "X jogadores".
        *   **Conector:** Ícone `ArrowRight` (Lucide) ou `MoveRight` em um círculo centralizado entre os cards.
        *   **Card Entrando (Direita):** Fundo Glass escuro com tintura verde/azul.
            *   Ícone: `LogIn` (Lucide) em badge pequeno.
            *   Texto: "ENTRANDO" + Nome do Time.
            *   Contador: "X jogadores".

3.  **Listas de Jogadores (Pills/Badges)**
    *   **Entrando em Quadra:** Lista de jogadores do time entrante.
        *   Estilo Pill: `bg-white/5 border border-white/10 rounded-full`.
        *   Badge Número: Fundo ligeiramente mais claro ou cor do time.
    *   **Jogadores Emprestados (Se houver):** Destaque em Amarelo/Amber.
        *   Card ou área separada com aviso sutil em itálico.

4.  **Sistema de Botões (Footer)**
    *   Organização hierárquica em 3 níveis visuais.

### 3. Definição de Estilos (Tailwind CSS)

#### Cores e Glassmorphism
Utilizar as variáveis CSS existentes e classes utilitárias para o efeito "Glass":
*   **Glass Padrão:** `bg-[#1e2030]/80 backdrop-blur-md border border-white/10`
*   **Tintura Vermelha (Saindo):** `bg-rose-500/10 border-rose-500/20 text-rose-100`
*   **Tintura Verde (Entrando):** `bg-emerald-500/10 border-emerald-500/20 text-emerald-100`

#### Sistema de Botões (3 Níveis)

*   **Nível 1 (Ação Principal - "PRÓXIMO"):**
    *   Largura total (`w-full`), Altura grande (`h-14` ou `h-16`).
    *   Background: Gradiente Primário (`bg-gradient-to-r from-indigo-600 to-purple-600`).
    *   Texto: Uppercase, Bold/Black, Tracking-widest.
    *   Shadow: `shadow-lg shadow-indigo-500/30`.

*   **Nível 2 (Ações Secundárias - "ENVIAR" & "DOWNLOAD"):**
    *   Layout: Grid 2 colunas (`grid-cols-2 gap-3`).
    *   Estilo: Ghost/Glass (`bg-white/5 hover:bg-white/10 border border-white/10`).
    *   Altura: Média (`h-12`).
    *   Ícones centralizados ou ao lado do texto.

*   **Nível 3 (Ações Terciárias - "DESFAZER" & "ZERAR"):**
    *   Layout: Grid 2 colunas.
    *   Estilo: Minimal Ghost (`bg-transparent hover:bg-white/5 border border-transparent hover:border-white/10`).
    *   Texto: Menor, cor mais apagada (`text-slate-400`).

## Decisões Técnicas

1.  **Motion Design (`framer-motion`):**
    *   Manter a entrada suave do modal (`AnimatePresence`).
    *   Adicionar `staggerChildren` para a entrada sequencial dos elementos (Header -> Rotation -> Buttons) para uma sensação mais premium.

2.  **Responsividade:**
    *   Utilizar Flexbox (`flex-col`) com `flex-1` na área de conteúdo (scrollável) para garantir que o Header e o Footer fiquem sempre visíveis se possível, ou rolem graciosamente em telas muito pequenas.

3.  **Componentes Reutilizáveis:**
    *   Não criar componentes novos em arquivos separados por enquanto (Badges/Pills serão estilizados inline com Tailwind para manter a coesão no arquivo, dado que são específicos deste contexto).

4.  **Lógica de Dados:**
    *   Manter o uso de `state.rotationReport` e `transferAnalysis`. A lógica de negócio não muda, apenas a apresentação.
